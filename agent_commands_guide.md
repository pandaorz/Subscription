# Antigravity Agent 執行指令指南 (15 種常見情境)

本指南提供 15 種針對 `subscriptions.md` 的操作指令，可直接於 VS Code 的 Gemini Chat 或 Agent 模式中使用。

---

## 📂 第一部分：狀態校對與日常維護

### 1. 全域狀態同步
* **指令**：`Agent，請根據今天的日期，重新核對 subscriptions.md 中所有項目的狀態。記得依據每項自訂的 N 值更新 🟢、🟡、🔴 圖示。`

### 2. 更新特定支出
* **指令**：`我剛繳完 [台灣電力] 帳單，金額為 1,850 元，下次扣款日更新為 2026-04-15。請更新表格並在 Change Log 紀錄。`

### 3. 調整預警天數 (N)
* **指令**：`將 [Perplexity Pro] 的預警 N 改為 10 天，因為我需要更長的時間決定是否續約。更新後請重新判斷其狀態。`

### 4. 切換預警開關
* **指令**：`我決定將 [iCloud] 改為手動管理，請將其「啟用預警」改為 Yes，並設定 N=5。`

---

## 📂 第二部分：財務審核與成本優化

### 5. 月度支出分析
* **指令**：`請統計目前「每月」的總支出，並分別列出 [Tech]、[Utility] 與 [Life] 各類別的佔比。`

### 6. 年度預算估算
* **指令**：`根據目前的清單，幫我試算未來一整年的總支出（含隔月繳的公共事業費用估計）。`

### 7. 冗餘服務偵測
* **指令**：`以「家庭財務主管」視角，分析 [Tech] 類別中是否有功能重疊的服務，並提出裁撤或整合建議。`

### 8. 運動成本轉向分析
* **指令**：`如果 [羽球臨打費] 每個月發生超過 4 次，對比 [羽球運動費] 的固定月費，哪種方案較具成本效益？`

---

## 📂 第三部分：資安管理與風險盤點

### 9. 帳號用途盤點
* **指令**：`列出所有 [Tech] 類別的「帳號用途」，檢查是否有已經不再使用的開發測試工具。`

### 10. 支付安全審查
* **指令**：`以「資安工程師」視角，檢查清單中哪些項目可能涉及國外交易或自動扣款，並提醒我確認是否已開啟該帳戶的二階段驗證 (2FA)。`

### 11. 敏感資訊掃描
* **指令**：`掃描 subscriptions.md，確認我是否不小心在備註或用途欄位留下了疑似密碼或 API Key 的資訊。`

---

## 📂 第四部分：生活與行政決策

### 12. 繳費高峰預警
* **指令**：`檢查未來 14 天內，所有「啟用預警: Yes」且狀態可能變為 🟡 或 🔴 的項目，幫我整理成一張待辦清單。`

### 13. 季節性變動提醒
* **指令**：`目前進入夏季/冬季，請提醒我關注 [Utility] 類別中電費或瓦斯費的異常波動風險。`

### 14. 匯率變動試算
* **指令**：`若美金匯率從 31 升至 33，請幫我重新計算所有 [Tech] 類別（假設部分為美金計價）的台幣支出價差。`

### 15. 新增項目模版化
* **指令**：`我要新增一個 [服務名稱] 到 [類別]，用途是 [用途]，月繳 [金額]，扣款日是 [日期]，N=7，預警開啟。請直接插入表格正確位置並排序。`

---

## 💡 使用小技巧
- **連續指令**：您可以合併指令，例如：「更新電費後，順便幫我重新計算月總支出。」
- **強制風格**：如果 Agent 開始客套，可以提醒它：「請遵循 instructions.md，保持專業嚴謹，直接給出分析。」
